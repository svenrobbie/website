# [c14h19no2.nl](https://c14h19no2.nl) – Cybersecurity Portfolio

Dit is de broncode van mijn persoonlijke portfolio. De website fungeert als mijn digitale commandocentrum en weerspiegelt mijn transitie van de biomedische wereld naar **Cybersecurity**.

## 🛡️ Over het Project

In tegenstelling tot standaard portfolios, is deze site ontworpen met een **CrowdStrike-inspired esthetiek**: gefocust op snelheid, precisie en een tactische interface. Het dient als bewijs van mijn vermogen om klinische precisie (uit mijn achtergrond als biomedisch analist) te combineren met digitale defensie.

### Kernpunten:
- **Focus:** Defensive Operations, Systems Hardening & Secure system design.
- **Identiteit:** Cybersecurity Engineering Student (ex-Biomedisch Analist MBO 4).
- **Design:** High-contrast tactical UI met een focus op leesbaarheid en technische autoriteit.

## 🏗️ Infrastructuur & Hosting

Deze website wordt gehost via GitHub Pages en de volledige broncode is open source beschikbaar op GitHub. Hierdoor blijft de infrastructuur transparant, reproduceerbaar en eenvoudig te onderhouden.

**Hosting:** GitHub Pages (statische hosting).

**Code:**: Volledig open source en publiek beschikbaar via GitHub.

**Architectuur:** Statische HTML5/TailwindCSS stack voor maximale performance en een klein aanvalsoppervlak.

**Privacy:** Ontwikkeld met privacy-first principes; geen externe trackers of onnodige scripts.

## 🛠️ Technische Specificaties

- **Frontend:** Tailwind CSS voor een gestroomlijnde, responsieve interface.
- **Typografie:** Inter (Italic/Bold) & JetBrains Mono voor een authentieke "code-first" uitstraling.
- **Componenten:** 
    - Tactische scanline overlays.
    - Responsieve grid-systemen voor vaardigheden.
    - Geïntegreerde SVG-icons voor GitHub en LinkedIn.

## 📂 Inhoud van de Site

- **Operationele Status** – Mijn huidige voortgang binnen de cybersecurity opleiding.
- **Technische Vaardigheden** – Expertise in Linux (Debian/Arch), Security Development (Python/Rust) en Netwerkbeperking.
- **Operationele Historie** – Mijn achtergrond in de laboratoriumwetenschap en mijn ervaring met home-lab beheer (Docker/VLAN-segmentatie).
